# 数据中心字段说明抽屉需求文档

## 1. 背景

“营收到卡轻量核算工具”用于帮助交付、运营、财务人员把不同平台的经营收入、结算金额、账期和预计到卡日期解释清楚。当前工具解决了“怎么算”的问题，但用户在录入和查看结果时，仍会遇到大量字段口径问题，例如：

- 营业额、结算金额、入账金额、实收金额、到卡金额是否相同。
- T+1/T+3 是从营业日期、结算日期还是账单日期开始算。
- 平台佣金、服务费、退款、可提现余额分别影响哪一层金额。
- 财务中心账期与平台打款账期不是同一个概念。

已有的“食亨数据中心字段说明”页面提供了字段字典、资金流转解释、对账说明和 FAQ。现在需要把其中“字段字典”能力合并进核算工具，并保持当前工具轻量、专注、可快速使用的定位。

## 2. 目标

在营收到卡轻量核算工具中新增一个“字段说明抽屉”，承载全部数据中心字段解释，并支持快速查询、分类筛选和字段详情查看。

核心目标：

- 用户无需离开核算工具，即可查询数据中心字段解释。
- 支持按字段名、所属中心、业务场景、公式、常见误解进行全文搜索。
- 字段解释以知识库形式组织，不干扰主流程录入和结果查看。
- 后续可扩展为配置化字段库，便于持续新增字段。

非目标：

- 本阶段不做线上文档中心接入。
- 本阶段不做登录、权限、字段编辑后台。
- 本阶段不要求把资金流转图、FAQ、对账核实工具完整迁移进来。
- 本阶段不修改核算算法，只新增字段解释能力。

## 3. 用户角色

- 交付老师：向品牌解释收入什么时候到卡，以及差异原因。
- 品牌运营：理解不同平台收入、账期、结算字段的区别。
- 财务核对人员：核对实收、应收、未收、服务费、佣金等字段。
- 内部支持人员：快速查询字段口径，减少重复沟通。

## 4. 入口设计

建议在页面右上角增加一个全局入口按钮：

```text
字段说明
```

按钮样式沿用当前项目的 glassmorphism 风格：

- 半透明背景。
- 细边框。
- hover 时轻微高亮。
- 可使用 lucide-react 中的 `BookOpen`、`Info` 或 `Search` 图标。

入口位置建议：

- 桌面端：顶部操作区，与主题切换、重置按钮并列。
- 移动端：顶部操作区保留入口，抽屉打开后全屏展示。

后续增强入口：

- 表单字段旁增加小型说明按钮，点击后打开抽屉并自动定位相关字段。
- 结果卡片中增加“解释这个结果”入口，自动展示到卡金额、账期、结算金额等相关字段。

## 5. 抽屉形态

### 5.1 桌面端

右侧大抽屉，建议宽度：

```text
min(720px, 92vw)
```

布局结构：

```text
抽屉头部
- 标题：数据中心字段说明
- 副标题：搜索字段口径、公式、常见误解
- 关闭按钮

搜索与筛选区
- 搜索框
- 分类筛选 chips/tabs
- 匹配数量

内容区
- 左侧/上方：字段列表
- 右侧/下方：字段详情
```

如果宽度足够，抽屉内部可以使用两栏：

- 左栏：字段列表，宽约 42%。
- 右栏：字段详情，宽约 58%。

如果宽度不足，则上下排列：

- 上方字段列表。
- 下方字段详情。

### 5.2 移动端

移动端建议为全屏抽屉：

- 宽度 100vw。
- 高度 100dvh。
- 顶部固定搜索区。
- 字段详情以列表下方展开，或点击字段后进入详情视图。

移动端优先保证：

- 搜索框可见。
- 分类可横向滚动。
- 字段卡片高度紧凑。
- 关闭按钮始终可触达。

## 6. 信息架构

字段说明抽屉包含 4 个核心区域。

### 6.1 搜索区

搜索框 placeholder：

```text
搜索字段、口径、公式、场景，例如：入账、T+1、佣金、可提现
```

搜索范围：

- 字段名。
- 字段分类。
- 数据来源。
- 一句话解释。
- 公式/口径。
- 常见误解。
- 下钻建议。
- 标签。
- 相关字段。

搜索行为：

- 输入即时过滤。
- 忽略大小写。
- 支持中文关键词。
- 空关键词展示全部字段。
- 无结果时展示空状态。

### 6.2 分类筛选

首版分类建议：

- 全部。
- 营业中心。
- 财务中心。
- 资金/到账。
- 服务费/佣金。
- 余额/提现。
- 订阅/台账。
- 对账/异常。

分类可以由字段配置中的 `category` 和 `tags` 共同驱动。

筛选逻辑：

- 点击分类后，只展示该分类或包含该标签的字段。
- 搜索词与分类筛选同时生效。
- 分类 chip 显示当前分类下匹配数量。

### 6.3 字段列表

每个字段列表项展示：

- 字段名。
- 一句话解释。
- 分类标签。
- 业务标签。
- 是否与当前工具相关。

示例：

```text
入账金额
资金进入平台账户、分账账户或银行账户的金额，是资金侧口径。
营业中心 · 资金/到账 · 当前工具相关
```

推荐交互：

- 默认选中第一个匹配字段。
- 点击字段后右侧展示详情。
- 当前选中项有渐变边框或轻微内发光。
- hover 时轻微 scale，保持与现有任务卡风格一致。

### 6.4 字段详情

详情内容建议：

- 字段名。
- 所属分类。
- 标签。
- 数据来源。
- 口径说明。
- 计算公式。
- 常见误解。
- 示例。
- 相关字段。
- 在本工具中的用途。

详情展示模板：

```text
字段名
标签：营业中心 / 资金到账 / 当前工具相关

口径说明
...

计算公式
...

常见误解
...

示例
...

相关字段
营业额 / 结算金额 / 实收金额 / 到卡金额

在本工具中的用途
...
```

## 8. 字段数据结构建议

建议将字段说明拆成独立配置，避免写死在 JSX 结构里。

```ts
type FieldDefinition = {
  id: string;
  name: string;
  category: string;
  source: string;
  summary: string;
  formula?: string;
  misread?: string;
  example?: string;
  tags: string[];
  relatedFieldIds: string[];
  toolRelevance: "high" | "medium" | "low";
  toolUsage?: string;
  drill?: string[];
};
```

推荐文件：

```text
src/data/fieldDefinitions.ts
```

如果字段数量继续增长，可再拆成：

```text
src/data/fieldDefinitions/
- operatingCenter.ts
- financeCenter.ts
- settlement.ts
- index.ts
```

## 9. 搜索规则

首版可以使用本地前端搜索，不引入额外搜索库。

匹配字段：

```text
name
category
source
summary
formula
misread
example
tags
toolUsage
drill
```

排序建议：

1. 字段名完全匹配。
2. 字段名包含关键词。
3. 标签包含关键词。
4. 摘要/公式/误解包含关键词。
5. 当前工具相关性高的字段优先。

搜索结果反馈：

```text
共匹配 8 个字段
```

无结果空状态：

```text
没有匹配到字段
可以尝试搜索“入账”“佣金”“账期”“余额”“实收”
```

## 10. 视觉与交互要求

需要遵循当前项目风格和 AGENTS 指令：

- glassmorphism：半透明背景、backdrop blur、细边框。
- 深色主题优先，同时兼容浅色主题。
- 使用 CSS custom properties，不写死单一主题颜色。
- 使用 Inter 字体。
- 使用 Framer Motion 实现抽屉进入/退出动画。
- 字段卡片 hover 有轻微 scale 和内发光。
- 空状态需要有插画感，可以用 CSS 图形或简洁图标组合，不依赖外部图片。
- 搜索加载或未来异步加载时预留 skeleton shimmer。

动画建议：

- 抽屉遮罩 fade in/out。
- 抽屉从右侧 slide in/out。
- 字段列表项 layout animation。
- 字段详情切换 fade + slight translate。

## 11. 状态设计

需要覆盖以下状态：

- 抽屉关闭。
- 抽屉打开。
- 默认全部字段。
- 搜索中有结果。
- 搜索无结果。
- 分类筛选无结果。
- 字段详情未选中。
- 字段详情已选中。
- 移动端字段详情视图。
- 未来异步字段加载中的 skeleton。

## 12. 与当前工具的关系

首版仅做全局字段说明，不强制与表单联动。

建议预留上下文打开能力：

```ts
openFieldDrawer({
  query?: string;
  fieldId?: string;
  tags?: string[];
});
```

后续可用于：

- 点击“结算金额”旁说明，直接定位“结算金额”。
- 点击“账期天数”旁说明，直接定位“账期 / 平台账期天数”。
- 点击结果卡片说明，搜索“预计到卡日期 到卡金额 账期”。

## 13. 验收标准

功能验收：

- 页面存在“字段说明”入口。
- 点击入口后打开右侧字段说明抽屉。
- 抽屉可关闭，关闭后主页面状态不丢失。
- 抽屉内展示全部首批字段。
- 支持关键词搜索。
- 支持分类筛选。
- 搜索与分类筛选可叠加。
- 点击字段后展示完整详情。
- 无搜索结果时展示空状态。
- 深色/浅色主题下均可阅读。
- 移动端可正常打开、搜索、查看详情和关闭。

内容验收：

- 旧字段说明页中的 22 个字段均被迁移。
- 每个字段至少包含字段名、分类、来源、摘要、公式/口径、常见误解。
- 当前工具强相关字段有明确标记。
- 术语需要区分“平台打款账期”和“财务中心账期”。

工程验收：

- `npm run build` 通过。
- 不引入新的重量级依赖。
- 字段数据与 UI 组件解耦。
- 搜索逻辑可单独测试或至少便于单独验证。
- 不破坏当前 board/list/calendar 视图切换。

## 14. 风险与注意事项

- 字段字典数据量变大后，普通列表会变难扫，需要依靠分类、搜索和详情结构降低认知负担。
- “全部数据中心字段”可能持续扩充，首版需要先定义数据结构，避免后续重写。
- 一些字段在不同系统中同名不同义，需要在详情中明确“所属中心”和“数据来源”。
- 当前旧需求文档存在编码显示异常，新文档应保持 UTF-8。
- 字段解释属于业务口径，后续最好明确字段负责人和更新时间。

## 15. 待确认问题

这些问题不阻塞首版抽屉设计，但会影响后续内容治理：

1. “全部数据中心字段”的权威来源是旧 HTML 页面，还是还有线上表格/飞书文档。
2. 字段解释是否需要展示负责人、更新时间、适用系统版本。
3. 是否需要支持复制字段解释，用于发给客户或内部群。
4. 是否需要将字段字典导出为 Excel/Markdown。
5. 是否需要在核算表单字段旁做第二阶段上下文入口。

## 16. 推荐实施顺序

1. 抽离字段数据配置。
2. 新增字段说明抽屉基础组件。
3. 实现搜索、分类筛选和字段详情。
4. 接入全局入口按钮。
5. 补齐移动端布局。
6. 加入空状态和 skeleton 样式。
7. 执行构建和视觉检查。
